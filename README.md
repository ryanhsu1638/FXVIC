# 匯倬FXVIC 零用金使用紀錄（PWA）

深藍黑科技風、可安裝到手機主畫面的離線記帳 App。純靜態網頁，資料全部存在使用者手機的
IndexedDB 本機資料庫，不需要任何後端或付費服務。

## 檔案結構

```
fxvic-pwa/
├── index.html          App 主程式（UI + 邏輯）
├── manifest.json        PWA 安裝設定
├── service-worker.js    離線快取與版本管理
├── icons/
│   ├── icon-192.png
│   ├── icon-512.png
│   ├── icon-192-maskable.png
│   ├── icon-512-maskable.png
│   └── apple-touch-icon.png
└── README.md
```

## 部署到 GitHub Pages（完全免費）

1. 在 GitHub 建立一個新的 Public repository，例如 `fxvic-pwa`。
2. 把本資料夾內「所有檔案」（包含 `icons/` 整個資料夾）上傳到該 repo 的根目錄（或推送到 main 分支）。
3. 進入 repo 的 **Settings → Pages**。
4. 在 "Build and deployment" 底下，Source 選擇 **Deploy from a branch**，
   Branch 選擇 `main`（或你的預設分支）＋資料夾選 `/ (root)`，按 Save。
5. 等待約 1 分鐘，GitHub 會給你一個網址，格式為：
   `https://<你的帳號>.github.io/fxvic-pwa/`
6. 用手機瀏覽器打開這個網址即可使用，且可安裝成 App（見下方安裝說明）。

> 注意：因為使用相對路徑（`./`），無論部署在網域根目錄或子路徑（如 GitHub Pages 的
> `/repo名稱/`）都可以正常運作，不需要修改任何程式碼。

## 手機安裝方式

**Android（Chrome）**
- 開啟網址後，畫面「設定」分頁若出現「安裝到主畫面」按鈕可直接點擊；
- 或點瀏覽器右上角選單 → 「安裝應用程式」/「新增至主畫面」。

**iPhone（Safari）**
- 開啟網址 → 點下方分享圖示（方框加箭頭）→ 選擇「加入主畫面」。
- iOS 不支援 `beforeinstallprompt`，所以只能透過 Safari 手動加入，這是 Apple 系統限制，
  無法用程式自動觸發。

安裝後皆會以全螢幕 standalone 模式開啟（不顯示網址列），並使用你指定的 App Icon。

## 離線使用說明

- 首次開啟需要網路（下載 App 本體、字型與圖表/匯出用的函式庫）。
- 之後 Service Worker 會快取所有核心檔案與 CDN 函式庫，離線也能開啟 App、記帳、查看統計。
- 所有記帳資料存在 IndexedDB，離線新增/修改/刪除都會立即寫入手機本機資料庫，
  不需要網路即可完整操作記帳功能。
- 唯一例外：Google Fonts 字型檔在離線時可能無法載入（會自動退回系統字型，不影響功能）。

## 更新 App 版本（重要）

每次修改程式碼、重新部署後，請打開 `service-worker.js`，把最上面的：

```js
const CACHE_VERSION = 'v1';
```

改成 `'v2'`、`'v3'`……（遞增即可）。這樣使用者下次開啟 App 時，Service Worker 會自動
抓取新版本並清掉舊的快取，**但完全不會影響 IndexedDB 裡已經記錄的收支資料**，
因為快取（畫面/程式碼）與資料庫（記帳紀錄）是兩個獨立的儲存空間。

## 資料庫版本管理

`index.html` 內的 `DB_VERSION` 常數用於 IndexedDB 的 schema 版本控制。目前為 `1`。
未來如果需要新增資料表（例如多帳本），只要把 `DB_VERSION` 加 1，並在
`openDB()` 的 `onupgradeneeded` 內用 `if(!db.objectStoreNames.contains(...))`
新增新的 store，既有資料表與資料都會被完整保留，不會遺失使用者的歷史紀錄。

## 資料備份建議

- App 內「設定」分頁提供「資料備份」（匯出 JSON）與「資料還原」功能。
- 建議使用者定期備份 JSON 檔案，避免因清除瀏覽器資料、更換手機或解除安裝而遺失記錄。
- 若未來要加入雲端同步，可以在不影響現有 IndexedDB 架構的前提下另外擴充。

## 分享功能限制說明

- 「LINE」分享按鈕使用 LINE 官方文字分享連結，會帶入本月報表摘要文字，開啟 LINE
  讓使用者選擇聯絡人送出（純靜態網站沒有後端，無法自動夾帶圖片檔案給 LINE）。
- 「其他分享」按鈕使用瀏覽器原生 Web Share API（分享報表圖片＋文字），
  支援度依手機/瀏覽器而定，Android Chrome、iOS Safari 皆有支援；若裝置不支援，
  會自動改為下載圖片供使用者手動分享。
- 「存到相簿」在 Android 會直接下載圖片，iOS 會開啟圖片頁面供使用者長按儲存
  （iOS Safari 對網頁自動存相簿有系統限制，無法完全略過長按步驟）。

## 「手機沒辦法儲存，重開就歸零」問題排查（v2 更新）

這次更新修正並強化了以下項目：

1. **修正 Service Worker 安裝失敗的 bug**：舊版 `service-worker.js` 用
   `cache.addAll([...])` 一次快取所有檔案，只要清單中有任何一個檔案（例如
   maskable icon）遺失或路徑錯誤，整個安裝流程就會直接失敗，導致離線快取
   完全沒有生效。新版改成逐一快取、個別失敗互不影響，並補齊了原本缺漏的
   `icon-192-maskable.png` / `icon-512-maskable.png`。
2. **新增「儲存狀態診斷」**：App 內「設定」分頁最下方會顯示 IndexedDB 讀寫測試、
   目前紀錄筆數、是否為持久化儲存、目前是否以「主畫面安裝」模式執行、
   已使用的儲存空間。若資料又消失，請先來這裡截圖確認狀況。
3. **請求持久化儲存權限**：App 啟動時會呼叫 `navigator.storage.persist()`，
   降低系統在裝置空間不足時自動清除本 App 資料的機率（Android Chrome 有效，
   iOS Safari 目前不支援此 API，但不影響其他功能）。
4. **快取版本升級為 `v2`**：使用者下次開啟 App 時會自動套用以上修正。

### 最可能的真正原因：iOS「加入主畫面」與 Safari 分頁是「兩個獨立的儲存空間」

這是 iOS 最常見、也最容易誤會的行為，**幾乎可以確定就是你遇到的狀況**：

- 如果你在 **Safari 分頁**裡開啟網站、記了幾筆帳，資料存在「Safari 這個分頁專用」的儲存空間。
- 接著你「加入主畫面」變成一個 App 圖示，**這個主畫面圖示會使用另一個全新、獨立的儲存空間**，
  跟 Safari 分頁裡的資料完全不共用。從圖示打開 App 第一次，看起來就會像「資料歸零」，
  但其實資料沒有真的不見，只是你切換到了不同的儲存空間而已。
- 同樣地，如果你把主畫面圖示**刪除後再重新「加入主畫面」一次**，iOS 也會建立一個全新的
  儲存空間，等同於重新安裝一個乾淨的 App，先前的資料不會帶過來。

**建議測試方式（請照此順序操作一次，確認是否真的正常）：**

1. 先把手機上舊的 App 圖示、Safari 分頁快取都清掉，重新從 GitHub Pages 網址開啟一次。
2. 直接在 Safari 分頁中「加入主畫面」，建立圖示（**過程中不要用 Safari 分頁記帳**）。
3. 之後**都只用主畫面上的圖示**打開 App，不要再用 Safari 分頁開同一個網址。
4. 用快捷鍵新增 1-2 筆紀錄。
5. 完全關閉 App（在多工列表上滑掉），**不要刪除主畫面圖示**，重新從圖示打開。
6. 到「設定」分頁執行「儲存狀態診斷」，確認顯示「✅ 目前以主畫面安裝模式執行」且紀錄筆數正確。

如果照上述步驟資料仍然消失，請把「儲存狀態診斷」的畫面截圖提供給我，我可以再進一步排查
（例如該手機的無痕模式、第三方清理 App、或是儲存空間不足被系統強制清除等情況）。

無論如何，**強烈建議養成定期使用「資料備份」匯出 JSON 的習慣**，這是唯一不受瀏覽器儲存政策
影響、100% 保證資料不遺失的方式。
